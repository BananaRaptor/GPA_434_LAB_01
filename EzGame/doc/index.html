Bibliothèque EzGame

Introduction

La bibliothèque EzGame permet de créer facilement une application graphique rudimentaire.

Ce document contient la documentation de l'Interface de Programmation de l'Application (IPA ou API en anglais).

EzGame a été créé avec la bibliothèque SFML.

Sommaire

En bref, la bibliothèque permet :

  • la création d'une application graphique
  • de connaître :
    • l'état du clavier
    • le temps écoulé
  • d'afficher à l'écran :
    • une couleur uniforme sur toute la surface
    • des textes
    • des cercles
  • d'exécuter une boucle principale personnalisée - tourne en boucle et appelle deux fonctions de contrôle :
    1. produire la logique de l'application selon le temps écoulé et l'état du clavier
    2. réaliser l'affichage requis

EzGame a été conçue pour créer facilement une application graphique rudimentaire. L'objectif n'étant pas de passer beaucoup de temps à apprendre un outil complexe ou de mettre l'emphase sur les aspects cosmétiques.

Architecture

EzGame est divisé en 3 parties conceptuelles :

  • le noyau de l'application est constitué de 4 classes :
    • ezgame::Application : le moteur principal
    • ezgame::Keyboard : le clavier
    • ezgame::Timer : une métrique de temps
    • ezgame::Screen : l'écran
  • le moteur graphique est constitué de
    • quatre classes :
      • ezgame::Vect2d : un vecteur mathématique en 2d (utilisé aussi comme position)
      • ezgame::Color : une couleur
      • ezgame::Text : un texte
      • ezgame::Circle : un cercle
    • une énumération :
      • ezgame::Alignment : la disposition relative d'un objet à afficher
  • une classe utilitaire :
    • ezgame::Random : outil de génération de valeurs aléatoires

Utilisation

L'utilisation de EzGame est basée sur le principe où le développeur donne un moteur de jeu à l'application. Le moteur de jeu attendu par la bibliothèque doit rigoureusement correspondre au concept ezgame::GameEngineRequirements. En bref, ce concept stipule que le moteur doit fournir six fonctions avec exactement les mêmes :

  • noms
  • signatures
  • valeurs de retour

Un développement avec EzGame se fait en 4 grandes étapes :

  1. créer son moteur de jeu
  2. créer une fonction WinMain
  3. instancier un objet ezgame::Application
  4. exécuter la fonction ezgame::Application::run en lui passant le moteur de jeu créé précédemment (attention, cette fonction est un template)

Voici un exemple de code présentant une utilisation minimale de EzGame. La classe développée se nomme GameEngine.

#include <EzGame>
// Déclaration et définition du moteur de jeu
// - affiche un texte et un cercle à l'écran
// - en appuyant sur la barre d'espacement, le cercle se déplace aléatoirement
// - en appuyant sur la touche 'escape', l'application termine
class GameEngine {
public:
GameEngine()
: mText("EzGame", 36.0f, ezgame::Vect2d(400.0f, 100.0f), ezgame::Color(0.15f, 0.25f, 1.0f), ezgame::Alignment::CenterCenter)
, mCircle(50.0f, ezgame::Vect2d(400.0f, 300.0f), ezgame::Color::Yellow, ezgame::Color::Red, 5.0f)
{
}
// Les 6 fonctions répondant aux exigences du concept ]>-----------------------------------------+
|
float width() const { return 800.0f; } // <-----[1]-+
float height() const { return 600.0f; } // <-----[2]-+
std::string title() const { return "EzGame demo"; } // <-----[3]-+
std::string iconFileName() const { return ""; } // <-----[4]-+
|
bool processEvents(ezgame::Keyboard const& keyboard, ezgame::Timer const& timer) { // <---+-[5]-+
if (keyboard.isKeyPressed(ezgame::Keyboard::Key::Space)) { // | |
mCircle.move(ezgame::Vect2d::fromRandomized() * 2.5f); // | |
} // | |
return !keyboard.isKeyPressed(ezgame::Keyboard::Key::Escape); // | |
} // ----+ |
|
void processDisplay(ezgame::Screen& screen) { // <---+-[6]-+
screen.clear(); // |
screen.draw(mText); // |
screen.draw(mCircle); // |
} // ----+
private:
ezgame::Text mText;
ezgame::Circle mCircle;
};
// Point d'entrée du programme
int WinMain()
{
ezgame::Application app; // < instantiation d'objet de la classe ezgame::Application
app.run<GameEngine>(); // < exécution de la fonction run -> démarre la boucle principale
return 0;
}
